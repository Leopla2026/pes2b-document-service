Documentos fiscais transformados em dados estruturados.

API para identificação, extração e integração de PDFs fiscais e contábeis, preparada para n8n, Microsoft 365 e demais sistemas do grupo P&S2B.

Autenticação

Os endpoints de processamento exigem uma chave enviada no cabeçalho X-API-Key.

X-API-Key: SUA_CHAVE_SEGURA
Armazene a chave em credenciais do n8n ou em variáveis de ambiente. Nunca exponha a chave em páginas públicas ou código-fonte.

Endpoints

GET/api/v1/health

Verifica a disponibilidade da API.

GET/api/v1/info

Retorna informações públicas e versão.

GET/openapi.json

Especificação OpenAPI.

POST/api/v1/pdf/extract

Processa um PDF no campo multipart file.

POST/api/v1/pdf/extract-batch

Processa até 10 PDFs no campo multipart files.

Exemplo com cURL

curl -X POST \
  'https://document.pes2b.com/api/v1/pdf/extract' \
  -H 'X-API-Key: SUA_CHAVE' \
  -F 'file=@documento.pdf;type=application/pdf'

Exemplo no n8n

Use um node HTTP Request com método POST, autenticação por header, body do tipo Form-Data e o arquivo binário no campo file.

URL: https://document.pes2b.com/api/v1/pdf/extract
Header: X-API-Key = SUA_CHAVE
Body Content Type: Form-Data
Campo: file
Tipo: n8n Binary File

Tipos de documentos

A API retorna o tipo detectado em processamento.documentType. Entre os tipos atualmente suportados estão:

DAS
DECLARACAO_PGDAS
RECIBO_PGDAS
EXTRATO_PGDAS
RELATORIO_SIMPLES
DECLARACAO_FATURAMENTO
COMBINADO_DECLARACAO_RECIBO_PGDAS

Rastreabilidade

Todas as respostas incluem o cabeçalho X-Request-Id. Guarde esse identificador nos logs do n8n para facilitar suporte e auditoria.

Erros comuns

401API_KEY_MISSING

O cabeçalho X-API-Key não foi enviado.

403API_KEY_INVALID

A chave enviada não é válida.

400INVALID_FILE

Arquivo ausente, inválido ou fora do formato aceito.

413FILE_TOO_LARGE

Arquivo acima do limite configurado.

500INTERNAL_ERROR

Erro inesperado. Informe o X-Request-Id ao suporte.
